# デフェクトログテンプレート（Defect Log Template）

省略用語（RACI, KPI, ADR, DDL, SLO, QA, PM, TRK, EX）は [../../../shared-references/glossary.md](../../../shared-references/glossary.md) の『略語・日本語対応表』を参照してください。


本テンプレートは、不具合対応全体の監査最小セット（記録に必須の項目）を定義します。

## 利用方法
- **対象**: 全14段階の実施内容を記録（docs/skill-logs/ ログファイルに append-only で記入）
- **出力形式**: Markdown
- **記録基準**: 段階完了ごと、かつ重要な決定ポイント（ゲート条件）で記録
- **参照優先順位**: 本テンプレート > individual Phase logs

---

## ログファイル基本構成

```
docs/skill-logs/defect_repair_${CATEGORY}_${DATE}.md

例:
docs/skill-logs/defect_repair_DB_20260327.md
docs/skill-logs/defect_repair_IPC_20260325.md
```

### ファイル名規則
- `CATEGORY`: 不具合カテゴリ （DB / IPC / Performance / UI / BusinessLogic など）
- `DATE`: 対応開始日（YYYYMMDD）

### Append-Only 運用ルール
1. 新規対応開始時に新ファイル作成
2. 段階完了ごとに内容を **末尾に append**
3. 既存記録の削除・上書きは禁止
4. 誤記修正は「訂正: …」として新たにセクション追記

## ログ整理のルール

### 追記時のルール（Append-Only）
1. **追記のみ** — 既存記録の削除・修正をしない
2. **タイムスタンプ付き** — 各セクションに「段階番号＋日時」を記載
3. **承認ステータス明示** —各ゲート時点で「未承認」「承認済」を明記

### 誤記発見時

```markdown
## 訂正: 段階4 の根拠ファイル（2026-03-27 11:15:00）

**誤記**:
- 記載：根拠ファイル = TableDefinition.sql

**訂正**:

**理由**: 正確なファイルパスを確認
```
